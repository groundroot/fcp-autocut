# 베타 테스트 작업 환경 안내 — 소스만 담은 새 프로젝트로 작업해주세요

> 🎯 **요약**: 작업 중이던 타임라인을 그대로 앱에 넣지 마시고,
> **자막화할 소스(메인 스토리라인 클립)만 새 프로젝트에 담아서** 그 프로젝트를 드래그해주세요.
> **커넥티드 클립(b롤·자막·어저스트먼트 레이어 등)이 없는 상태**여야 합니다.

## 권장 작업 순서

1. 파이널컷에서 **새 프로젝트**를 만듭니다 (원본과 같은 해상도·프레임레이트).
2. 자막화가 필요한 소스 클립만 복사해서 새 프로젝트의 **메인 스토리라인**에 붙여넣습니다.
3. 타임라인에 **커넥티드 클립이 남아 있지 않은지** 확인합니다.
4. 이 새 프로젝트를 FCP AutoCut에 드래그해서 실행합니다.
5. 결과물(자막 타임라인 / 무음컷 압축 타임라인)을 돌려받아 원래 작업 프로젝트에 활용합니다.

## 왜 이 방식인가요?

### 기존 방식의 문제 — export 대기 시간

지금까지는 mp3를 추출하거나, 어느 정도 편집된 상태의 mp4를 **내보내기(export)한 뒤**
자막화를 진행하셨을 텐데, 이때 영상 길이만큼 export 시간이 발생합니다.
**소스만 담은 새 프로젝트를 직접 드래그하면 export 과정 자체가 사라져서** 이 시간이 단축됩니다.

### 편집 중인 타임라인을 그대로 넣으면 안 되나요?

편집 중이던 타임라인을 그대로 내보내는 것이 가장 효율적인 방법인 것은 맞습니다.
하지만 베타 기간 동안은 아래 두 가지 이유로 지원하지 않습니다.

1. **FCPXML 버전 차이** — 파이널컷 버전에 따라 XML 구조가 달라 호환 문제가 생깁니다.
2. **무음컷 시 커넥티드 클립 재조정 문제** — 무음 구간을 잘라낼 때, 그 구간 위에 놓인
   커넥티드 클립을 유지하면서 위치를 재조정하는 로직은 파이널컷 구조상 고려해야 할
   조건이 너무 방대해서 버그를 피할 수 없습니다.

그래서 안정적인 결과를 위해 **"소스만 담은 새 프로젝트"로 사용 방식을 변경**했습니다.
안정화 이후 개선을 검토하겠습니다.

## 한 줄 정리

> **편집본 export ❌ → 소스만 새 프로젝트에 담아 드래그 ⭕**

문의나 문제가 있으면 앱 하단의 **[피드백]** 버튼으로 알려주세요. 🙏
